# Enterprise Facilities Operations Diagnostic & Process Redesign

## Overview
This project simulates an end-to-end operational diagnostic of multi-site commercial facilities operations.  
The goal is to identify SLA breach drivers, analyze operational inefficiencies, and redesign workflows using data-backed business analysis.

This case study is modeled on enterprise Commercial Real Estate (CRE) environments such as corporate campuses, malls, and office towers.

---

## Objectives
- Diagnose SLA failures using operational data
- Identify root causes across vendors, assets, and maintenance types
- Apply structured business analysis frameworks
- Redesign current-state workflows into future-state processes
- Deliver implementation-ready recommendations

---

## Tech Stack
- **SQL (MySQL)** – Data cleaning and analytical modeling  
- **Power BI** – Operational diagnostics and visualization  
- **Visio / Lucidchart** – Process mapping (current vs future state)  
- **Business Analysis Frameworks** – Pareto, 5 Whys, failure mode mapping  

---

## Dataset
Semi-synthetic facilities operations dataset (~2,000 records) representing:
- Multi-site CRE portfolios
- Vendor-managed maintenance
- SLA-driven service operations

Key fields:
- Issue category and asset criticality
- Vendor tier and performance indicators
- SLA targets and breach flags
- Maintenance type (Preventive vs Reactive vs Emergency)
- Root cause tags for diagnostic analysis

---

## Key Analysis Areas
1. SLA breach concentration analysis  
2. Vendor performance diagnostics  
3. Preventive vs reactive maintenance impact  
4. Root cause clustering and Pareto insights  
5. Operational bottleneck identification  

---

## Process Redesign
The project includes:
- Current-state workflow mapping  
- Bottleneck identification  
- Future-state process redesign  
- KPI framework for SLA governance  
- Vendor management improvements

## Deliverables
